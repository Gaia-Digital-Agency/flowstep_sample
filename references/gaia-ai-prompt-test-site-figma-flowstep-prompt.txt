## Tech stack constraint
Designs must fit the PostgreSQL, Headless Payload CMS, Python Flow Scripting, ViteJS, ReactJS, TailwindCSS, NodeJS stack.
Component patterns, form behaviour, and routing should align with this stack:
  - React functional components, hooks, and React Router conventions
  - Tailwind utility classes — design tokens expressible as Tailwind config
  - Vite-friendly asset handling (no heavy build-time transforms)
  - Content modelled in Headless Payload CMS (collections, globals, blocks)
  - Forms submit via Node API endpoints; long-running flows handed off to Python Flow scripts
  - Data persisted in PostgreSQL via Payload

## Component library (use exclusively)
- Use **shadcn/ui (Radix + Tailwind)** for every interactive primitive.
- Do not invent new button / input / dialog primitives — extend the library above.
- Compose page sections from these primitives only; name custom blocks consistently (e.g. `HeroBlock`, `FeatureGrid`, `ContactBlock`).
